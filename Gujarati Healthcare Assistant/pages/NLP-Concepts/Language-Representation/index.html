Word Embeddings & Contextual Embeddings

Dense vector representations that capture semantic similarity between Gujarati medical terms — from static word-level embeddings to dynamic context-aware representations that adjust meaning based on surrounding text.

network_node

✅ Word Embeddings (Word2Vec, GloVe, FastText)

Word embeddings map each word to a fixed-length dense vector. Word2Vec (CBOW/Skip-gram) learns from local context windows, GloVe leverages global co-occurrence statistics, and FastText adds character n-gram subword information — critical for Gujarati morphology.

તાવ → શરદી (GloVe: 0.74)

✅ Contextual Embeddings (ELMo, BERT, GPT)

Unlike static embeddings, contextual models like ELMo, BERT, and GPT produce different vectors for the same word depending on its surrounding context. ELMo uses bidirectional LSTMs, BERT uses masked language modeling, and GPT uses autoregressive generation — each yielding rich context-aware representations.

Contextual Variability for "તાવ" (fever)
"દર્દીને તાવ છે" → symptom vector
"દવાથી તાવ ઉતર્યો" → treatment response vector
"બાળકને તાવ આવે છે" → pediatric context vector

High-Dimensional Vector Space

Each Gujarati medical term is mapped to a vector in continuous space. Static embeddings give one vector per word; contextual embeddings produce a unique vector per occurrence. Semantically related terms cluster together regardless of the approach.

Static vs Contextual Embeddings

Static embeddings (Word2Vec, GloVe, FastText) assign one fixed vector per word — efficient but blind to polysemy. Contextual embeddings (ELMo, BERT, GPT) produce dynamic vectors that change with context, capturing phenomena like medical term ambiguity.

GloVeStatic, global co-occurrence

Subword Power: FastText for Gujarati

FastText represents each word as a bag of character n-grams, making it exceptionally powerful for Gujarati where morphological variations and compound words share subword units. Rare or unseen medical terms still get meaningful embeddings.

Dense vs. Sparse Representations

Unlike sparse TF-IDF vectors, dense embeddings (both static and contextual) capture syntactic and semantic relationships that sparse methods cannot express. Static embeddings provide a fast, lightweight baseline while contextual embeddings offer deeper understanding.

Pre-trained Gujarati Embeddings

We leverage publicly available pre-trained FastText embeddings for Gujarati (Common Crawl + Wikipedia) for static representations, and multilingual BERT/XLM-R for contextual embeddings, fine-tuned on a curated corpus of 100K+ Gujarati clinical documents.
